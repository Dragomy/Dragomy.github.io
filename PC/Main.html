Completely switching to Linux. How is it after two weeks? (Posted 11/06/2024, modified 12/06/2024)

The final push came when Microsoft announced their recall feature. After updating my system, Edge was back on my desktop. Sigh.

On my laptop, I switched from Debian to Manjaro because Debian seemed a bit clunky. So, Manjaro it is. I installed it on my PC and, wow, everything seems to work. Time to start downloading my Steam library.

But where is my secondary drive that I use for all my games? Oh, it’s simply not mounted. Error: wrong fs type, bad option, bad superblock. Ok, what's a fs? I asked a friend for help.

His response: "Was für ein FS ist drauf?" ("What FS is on it?"). I realized he meant filesystem. That makes more sense.

Oh, it could be NTFS, the Windows filesystem. Let's use GParted and change it.

Hmm, when I try saving changes from GParted, the window crashes. Well, that’s bad.

I started GParted on my USB stick with the big RAM option. It works... great, first problem solved! Now to mount the drive... yes, that worked too! Woohoo!

Now that my library was downloaded, I tried starting Satisfactory. The Vulkan shader loading window popped up and then it closed. I tried some other games and... the same thing happened. I went through forums and found out I needed to install through Flatpak. Okay, let me try that. It works.

Satisfactory starts and crashes in the loading screen and cannot be closed, not by System Monitor and I don't know the terminal command. Elden Ring? The BattleEye window starts and that’s it. Dark Souls has 30fps in the EULA screen. But hey, the original Half-Life works.

Ok, I’m running open source drivers with an Nvidia graphics card. Maybe other drivers? I install the drivers, nothing happens.

Ok, maybe a distro that has native Steam/Proton support.

I switched to something more familiar: Pop!_OS (Ubuntu) with KDE Plasma.

No real problems with Pop!_OS. There were some things about gaming. Dark Souls wouldn’t start in fullscreen, but switching to Proton 7.06 fixed that. Thanks to a friend and ProtonDB.

Discord doesn’t support stream audio on Linux (in the app, not the web version), which is a bit sad, but I can live with that.

Now I've experienced but a glimpse of what: "Linux is free if you don't value your time" means. But hey, I like it.
